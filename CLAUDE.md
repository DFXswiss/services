# Claude Code Guidelines for DFX Services

## Git Workflow

**WICHTIG: Niemals direkt in `develop` oder `main` pushen!**

Immer:
1. Feature-Branch erstellen: `git checkout -b feature/beschreibung`
2. Änderungen committen
3. Push zum Feature-Branch: `git push -u origin feature/beschreibung`
4. Pull Request erstellen via `gh pr create`

## Commit Messages

- Keine AI-Erwähnungen in Commits (kein "Generated by Claude", etc.)
- Conventional Commits Format: `feat:`, `fix:`, `refactor:`, `test:`, `chore:`
- Professionelle, technische Beschreibungen

## E2E Tests

- Synpress Tests für MetaMask-Integration: `playwright.synpress.config.ts`
- Standard Playwright Tests: `playwright.config.ts`
- Screenshots in `e2e/screenshots/baseline/`
- Test-Wallet Seed in `.env.test` (TEST_SEED)

## Projekt-Struktur

- Frontend: React/Next.js
- E2E Tests: Playwright + Synpress für Wallet-Tests
- Port: 3001 (localhost)
